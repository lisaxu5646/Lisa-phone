# 星露谷 · 小克陪玩 MCP 备忘

> 记录日期：2026-07-17。状态：方向已确认，尚未安装模组或连接正式存档。

## 目标

让小克通过本机桥接看见 Lisa 正在玩的《星露谷物语》，先成为能理解当日处境、共同记住经历的陪玩者；验证稳定后再逐步获得少量、安全、可撤销的游戏动作。最终可研究独立 2P/3P 同伴身体，但不把“作弊工具很多”等同于“已经拥有一个可靠队友”。

## 推荐三阶段

### 1. 只读陪玩（首发）

- 读取日期、时间、天气、地点、体力、背包、任务、附近人物、农田和作物状态。
- 小克可以回答“今天还漏了什么”“接下来去哪”，也能基于当下状态自然聊天。
- 每晚睡觉结算时，只生成“今日共同经历”的事件候选；Lisa 预览确认后才进入现有事件层/记忆库。
- 游戏状态本身不整包塞进长期记忆，只保留有意义的第一次、共同选择、意外、承诺与关系事件。

### 2. 有限动作

- 第一批只考虑：跟随、走到指定地点、浇指定区域。
- 所有动作经过白名单、范围限制、超时停止和卡住恢复。
- 花钱、送礼、丢弃/出售物品、传送、改时间、生成物品、改关系值默认禁止。
- 写动作必须有独立开关；一键退回只读。

### 3. 独立同伴身体

- 独立精灵、地图实体、寻路、动作队列、使用工具、下矿/战斗/钓鱼。
- 这一阶段不是普通 MCP 配置，需要 SMAPI C# 模组承担实体和游戏循环，MCP 只提供感知与高层行动接口。
- 先用测试存档验收；正式存档必须每日备份，且不得让模型直接调用任意调试命令。

## 预定架构

`Stardew Valley + SMAPI 模组 → 本机只读/白名单桥 → CC/小克 → Lisa-phone 事件候选与记忆表`

- 星露谷桥只暴露最小必要状态，不接触 Supabase 密钥。
- 小克仍从现有角色档案与权威记忆表取得人格连续性。
- 游戏事件进入候选箱，不让模组直接写正式人格或长期记忆。
- MCP 写工具与记忆写工具分权；能记住不代表能操控游戏，能操控游戏也不代表能改人格。

## 开工前要确认

- Lisa 的游戏平台、系统与游戏版本；Mac/Steam 路径和 Windows 不同。
- 是否已经安装 SMAPI，以及现有模组清单。
- 先复制一个测试存档。
- 审核候选 StardewMCP/同伴项目的源码、许可证、最近维护状态与实际工具列表，不能只看展示视频。

## 言秋合并笔记（2026-07-18，与 Codex 三阶段拼图）

从 awesome-ai-companion 挖到 **NagiBridge**（https://github.com/anqinou-art/NagiBridge）：
SMAPI 模组，localhost HTTP API（7842=房主 / 7843=农场工），带 get_state/move_to/use_tool/farm/mine/harvest 等 17 工具 + 游戏内聊天 + **Claude Code Channel 模式**。Mac 官方支持，Releases 有现成 dll。

**与三阶段的对应**：
- 阶段1 只读陪玩：只用它的 get_state + 聊天 pull/push，写动作一律不调——零自研即可开跑。
- 阶段2 白名单动作：不直连——由言秋写一个本机小闸门脚本夹在中间，只放行「跟随/走到/浇水」，花钱送礼丢弃传送全部拦截，独立开关一键退回只读。
- 阶段3 独立身体：合作农场开农场工=现成的同伴身体（7843 口），不用自研 C# 实体框架。
- 事件入库仍按 Codex 原案：每晚只出「今日共同经历」候选，Lisa 过目才进事件层；桥不碰 Supabase 密钥。

**⚠️开工前**（Codex 的确认清单照办 + 补充）：NagiBridge 仓库未标 license 且要过一遍源码（尤其网络面是否只绑 localhost）；先测试存档；她的平台/版本/SMAPI 现状待她提供。定档为「大档」活：攒结转日集中施工。
